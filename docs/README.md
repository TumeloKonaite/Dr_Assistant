# Documentation

This folder holds the detailed documentation that supports the root README.

## Guides

- [getting-started.md](getting-started.md): environment prerequisites, installation, and initial setup
- [running-the-project.md](running-the-project.md): how to use the CLI, API, frontend, Docker stack, and `Makefile`
- [data-and-artifacts.md](data-and-artifacts.md): knowledge-base inputs, generated KB artifacts, and pipeline outputs
- [post-consultation-workflow.md](post-consultation-workflow.md): full stage-by-stage explanation of the canonical workflow
- [testing.md](testing.md): automated test execution and workflow-focused test paths
- [testing-with-synthetic-dataset.md](testing-with-synthetic-dataset.md): how to test the repository with the synthetic consultation dataset

## When To Read What

- Start with [getting-started.md](getting-started.md) if you are setting up the repo for the first time.
- Use [running-the-project.md](running-the-project.md) if you want commands for local runs, API usage, or containers.
- Use [data-and-artifacts.md](data-and-artifacts.md) if you need to understand what is stored under `data/` or `outputs/`.
- Use [post-consultation-workflow.md](post-consultation-workflow.md) if you need the orchestration and contract flow.
- Use [testing.md](testing.md) if you want the basic test commands and evaluation paths.
